    Functions are first class objects and have properties and methods.
    1. length property: nameFunction.length//returns number of parameters the function requires.
    2. Call method:
      const param = {name: 'Clark'};
      nameFunction(){return `Hello, my name is this.name`;}
      nameFunction.call(param)
      //the call(param) looks inside the param supplied as parameter looks inside the const param for any name value pair with the same name as this.name and replaces it with this.name with the value.
      If the nameFunction requires more parameters those are supplied as arguments following the first parameter.
      Example: function sayHello(greeting='Hello'){
      return `${ greeting }, my name is ${ this.name }`;       sayHello.call(clark, 'How do you do');
       //output: 'How do you do, my name is Clark'

    3. Apply method
      apply() // works same as the call method except the arguments after the first argument this

      Example: nameFunction.newProperty = newPropertyValues // this adds newPropertyValues with the name of newProperty to the nameFunction

  • Immediately Invoked Function Expression, IIFE
    This is a function that is invoked as soon as it was created. To invoke a function we add parenthesis at the end of the function nameFunction
    Example: nameFunction();// this invokes the nameFunction. However an IIFE does not need to have a name and instead of the name of the function the whole expression of the function takes place of the nameFunction; the parenthesis remain to act as the invocation and the whole expression is placed inside a parenthesis as well.
    syntax is (function() { functionExpression}());
    1. Temporary variables: Variables defined inside an IIFE as have a local scope and are unavailable outside the IIFE.
    2. Initialization code: IIFE is used to set up codes that run once on initialization and not needed thereafter. Code blocks do the same as well {code block Expression }
    3. Safe use Strict mode: it is recommended to use strict mode within IIFEs. So the mode works within only code written by you rather than applying to all code especially if working in a team.

  • Functions can redefine themselves by using the assignment statement outside the function expression itself without parenthesis
    Or they can rewrite itself by using the assignment statement within the function expression itself.
  • Init-Time branching: can be used to check browser functionality at runtime and then implement a fucntionality according to availbility of the functionality.
  • Recursive Function: calls itself repeatedly for as long as a condition is not met. Can be used in iterative processes. Example is the factorial of a number that will continue to run until the arguement is 1
  • Callback are functions passed as arguements to another function and then called inside the function they were passed to.
  • Higher order functions are functions that return other fucntions.
