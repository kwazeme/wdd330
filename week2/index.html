JNINJA: Arrays, Logic, and Loops

  • Some Array Properties and Methods:
    • array.length; //returns the number of items in arrray.
      array.length = n; cuts the array lenght to n. Removes all elements beyond n, or adds blanks (undefined) items if n is larger than initial array length.
    • array.pop(); //removes last item from array.
    • array.shift(); // removes first element of arrray.
    • array.push("element"); // appends element to end of arrray and returns lenght of new array.
    • array.unshift("element"); // appends element to beginning of array and reurns the new lenght.
    • array.concat([nArray]); //Adds elements of nArray into array and assigns to a new array unless the assignment is specified.
    • Set{}; //is a collection of unique elements.
      const nSet = new Set(); // set constructor.
      nSet.add(element); // adds element into nSet only if element does not exist already in nSet.
      if element is in nSet already, operation is ignored.
    • Ternary operator syntax
      condition ? (//code to run if condition is true) : (//code to run if condition is false)

JNINJA: Functions

  • Functions can be treated like objects: they can be assigned to variables, stored in arrays or returned by other fucntions.
  • functions can be created with either of 4 methods:
    • Using function delaration: function nFunction() {//code block;}
    • Anonymously: variable = function(){ //code block;};
    • Using a constructor: variable = new Function("//code block";);
      constructor method creates a global scoped function and should be avaoided generally.
    • Using arrow notation: function = () => {//code block;}. The curly braces is not compulsory for single line of code.
  • Hoisting is the action of moving variables or function decalrations to the top of the current scope.
    To hoist a function just invoke it at the start of the scope or page.
  • CallBacks: This is a function passed as argument to another function.
