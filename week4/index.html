JNINJA: Forms

  • The basic syntax of a form is the form tag. Search Form Example
  • Legacy DOM uses a document.forms to return all a collection of all forms on the document. Each form is them accessible by using the index of the form counting from top of document to bottom as the forms appear in that order. example: document.forms[0] will return the first form on the document.
    Other ways of accessing form and its elements:
    1. document.getElementByTagName('form')[0] //gets the first form in document.
    2. document.forms.search; //gets the form named "search"
    3. document.forms['search']; // also gets the named form. use this method if form name has invalid characters.

      4. Use any of the above assigned to a variable, example "form" to define the form. Then use the below methods to access the form elements.
    1. const [input,button] = form.elements; //gets the input and button element of the form
    2. form.searchInput; //gets the named input element of form
    3. form['searchInput']; // same as but used for when invalid caharaters are present.
    4. form.submit(); //submits the form without an event trigger.
    5. form.reset(); // resets values of the form. Used with caution.
    6. <button type="reset"> Reset< button> // Also resets form inputs values.
    7. form.action='/another/url.ext //define the form url for processing.
  • Form Events: Some events trigged by forms input are:
    1. focus event; brings focus to active field. example: input.addEventListener('focus', () => alert('focused'), false)
    2. blur event input.addEventListener('blur', () => alert('blurred'), false);
    3. change event input.addEventListener('change', () => alert('changed'), false);

Object Oriented Programming

  • OOP is programming by creating objects that contain both data and fucntions modeled after real world representations fo objects.
    objects have both methods, functions and attributes.
  • There are three concepts in OOP: encasulation, Polymorphism and Inheritance.
  • Encasulation involves enclosing the programming logic wihin and making only methods and functionality available to the user.
  • Polymorphism: The use of same methods by different objects with individual objects being able to override certain methods or to have specific methods that are unique to them.
  • Inheritance: Means aready existing objects in the programming can have their methods and properties taken up by new objects created
  • JavaScript uses both classes and prototype as basis for creating objects.
  • new is the keyword for the constructor for creating instances of an object.
  • obj.constructor; //returns the constructor property that created the object.
  • static name() {}; are methods that are unique to the object and cannoot be inherited by new instances.
  • Create only class declarations that have initialization and shared prperties and methods.
  • Additonal methods and properties can be added after the fact:
    Example: objName.prototype.name = function() {}; //adds a new method to an existing objName. This method will be live and is inherited by all instances previouly created as well as new instances that will be created.
  • Unique properties or methods sepecific to an instance can be augemented using and assignment statement.
    Example: InstanceName.property = 'new PropertyValue';
  • Mixins is a way of adding properties and methods of some objects to another object without using inheritance.

Modern JavaScript Development

  • Use libraries to achieve methods and common task but carefully evaluate what you need to avoid bloat. Jquery and Lodash two most popular libraries.
  • npm and Yarn are common package managers. Installing anyone of them will also install the depencies and setting are saved in a package.json file.
  • Modules are self contained pieces of code much like a snippet that can be reused.
  • Before code is deployed, it should be concatenated into a single file, minified and compressed. The script tag should be placed just before the closing tag to ensure that all elements on the page have loaded before the script runs.
